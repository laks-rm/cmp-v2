
- Create the migration files
- Apply the schema to your database
- Prompt for a migration name (e.g., "initial_schema")


## Notes

- All timestamps are in UTC
- Comments are append-only (no updated_at, no delete)
- Audit logs are immutable (cannot be edited or deleted)
- Task codes auto-format as T-NNNNN (e.g., T-00001)
- Source codes auto-format as SRC-NNN (e.g., SRC-001)
- Evidence files stored in cloud storage (file_url field)
- Date fields use @db.Date for date-only columns
- Text fields use @db.Text for unlimited length

## Troubleshooting

**Migration fails:**
- Ensure PostgreSQL is running
- Check database credentials in .env
- Verify database exists: `psql -U lakshmibichu -d cmpv2 -c "SELECT version();"`

**Seed fails:**
- Ensure migrations are applied first
- Check for any existing data conflicts
- Drop database and recreate if needed: `npm run db:push --force-reset`

**Prisma Client errors:**
- Re-generate client: `npm run db:generate`
- Clear node_modules and reinstall: `rm -rf node_modules && npm install`
